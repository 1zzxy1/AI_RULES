# AI Decisions

## 决策记录

### 2026-06-14 | 使用 `.ai/` 作为跨 AI 公共工作台

- 背景：Codex、Claude、Gemini 等平台各有自己的规则入口，单个文件难以承载共享交接状态。
- 决策：平台入口文件只放稳定规则，项目状态统一放项目根目录 `.ai/`。
- 影响：不同 AI 可以通过同一组 Markdown 文件交接，降低上下文断裂。
- 回滚条件：如果某平台无法可靠读取隐藏目录，可改用 `AI/` 或 `AI_CONTEXT/`，并保留兼容说明。

### 2026-06-14 | RULES.md 采用「行为规则 + 协作协议」两部分结构

- 背景：Codex 初版 RULES.md 只覆盖跨 AI 交接协议；用户本机全局规则里还有大量编码行为规则（语言、编码原则、Git 纪律、路径、安全）需要沉淀进来。
- 决策：RULES.md 作为唯一权威来源，分两部分——第一部分行为规则、第二部分跨 AI 协作协议。
- 影响：单一文件即可表达「怎么写代码」和「怎么交接」，平台副本据此自包含派生。
- 回滚条件：若两部分耦合导致维护困难，可拆为 `RULES.md`（协议）+ `CODING_RULES.md`（行为）两文件。

### 2026-06-14 | 平台入口文件改为自包含

- 背景：初版平台文件仅含协议、指向 RULES.md；但用户场景是「单文件复制到各平台/各机器」，复制单文件时 RULES.md 未必同在。
- 决策：每个平台文件自包含完整规则（行为 + 协议），顶部标注权威来源为 RULES.md。
- 影响：可单文件复制即用；代价是改规则需同步多份副本（已在 README/adoption/CHANGELOG 写明流程约束）。
- 回滚条件：若同步副本成本过高，可改为「平台文件用 `@import` 引用 RULES.md」的薄入口模式。

### 2026-06-14 | 交叉分析采用「守门人对抗式过滤」防止规则膨胀

- 背景：本机有大量多来源提示词（Codex/Claude/Gemini/旧备份），直接合并会让通用规则臃肿，违背用户最高原则「极简优先」。
- 决策：用 agent 集群挖掘候选后，加一道保守的「守门人」逐条判定 belongsInUniversal/placement/minimalismRisk，拿不准就从严（降级为可选 profile 或拒绝）；15 候选只放行 5 条，且能并入现有条目就不另起。
- 影响：通用规则只吸收真正跨 AI 通用、非重复的增量；工具/编排/重工程偏好类一律不入核心。
- 回滚条件：若后续发现放行标准过严漏掉重要规则，可放宽到「跨 ≥2 源 + 不重复」即纳入。

### 2026-06-14 | 三个冲突项的用户裁决

- 注释语言：由「保持英文」改为「跟随代码库已有语言（无先例默认英文）」；变量名/commit/报错仍英文。
- 提交策略：以用户全局铁律「每次改动即提交」为权威，删除从 Codex 协议带入的「未经用户要求不主动提交」（曾存在于 RULES 第 9 条与 codex/generic 副本，属内部矛盾，已统一）。
- 既有文档：「不主动生成文档」细化为「不新建文档文件，但改动影响到的既有文档/注释同步更新，不让其变陈旧」。
- 回滚条件：任一条用户改主意时，先改 RULES.md 再同步四副本。

### 2026-06-14 | 全局安装与项目安装分离、机器差异用 profile 分层

- 背景：用户希望仓库成为每台机器 `~/.claude/CLAUDE.md`、`~/.codex/AGENTS.md`、`~/.gemini/GEMINI.md` 的通用来源。
- 决策：新增 `install-global-rules.{ps1,sh}`，与项目级脚本并列；全局脚本用「覆盖式同步 + 时间戳备份」（全局规则需随仓库更新，不能 skip-existing），机器差异放 `profiles/<machine>/` 作追加片段，不污染 platforms/ 通用核心。
- 影响：通用核心单一来源（platforms/），各机器重跑脚本即可拉取更新；个性化隔离在 profile。
- 回滚条件：若覆盖式同步风险过高，可改为先备份后交互确认或默认 `--dry-run`。

### 2026-06-14 | 蒸馏后采用「短核心 + 薄入口 + 偏好 profile」

- 背景：Mac 与 Windows 两轮迭代后，规则已覆盖完整但开始膨胀；平台自包含副本和强个人偏好会增加同步成本。
- 决策：`RULES.md` 压缩为跨平台最小核心；`platforms/` 改为薄入口；「结论先行 / 计划门禁 / 小步提交倾向」下沉到 `profiles/rainor-opinionated/`；`profiles/rainor-macos/` 保留为历史快照参考。
- 影响：通用规则更短、更稳定；个人工作流仍可通过 profile 追加；旧的「平台副本完整自包含」和「每次改动即提交是核心铁律」不再作为通用核心要求。
- 回滚条件：若某平台不能可靠访问 `RULES.md` 或薄入口效果差，可恢复为自包含副本，但应由脚本生成以避免手工漂移。
